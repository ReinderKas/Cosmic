# Bot Patch Notes v0.3

Compared against `master`.

This note focuses on player-visible bot behavior, commands, automation, and GM tooling. Internal refactors, test-only changes, local tooling, and documentation-only commits are omitted unless they changed bot behavior. The **Bug Fixes** section only lists issues that already existed on `master`; fixes for regressions introduced and later resolved on `experimental` are intentionally excluded.

## 1. New Features

- Added targeted follow commands so bots can follow a named party member or sibling bot instead of only the owner.
- Added anchored grind commands such as `farm here`, `grind here`, `sentry`, `camp`, `guard`, `post up`, and `anchor here` so bots can hold and farm from a fixed position.
- Added manual `fidget` behavior for small idle/social movement bursts.
- Expanded bot status chat commands with `exp`, movement-stat breakdowns, crit breakdowns, buff debug, skill-buff debug, and auto-pot debug output.
- Added owner/bot supply request flows for HP pots, MP pots, mixed pot needs, and bow/crossbow ammo shortages.
- Added ammo sharing and star/ammo offer handling for ranged bots.
- Added `fame me` and `fame <name>` bot chat commands, including normal daily/monthly fame-limit behavior.
- Added slot-based trade commands such as `trade <slot>`, `show me your <slot>`, and `show me your junk`.
- Added `trade trash` / `trade junk` so bots can hand over disposable, non-reserved equips separately from protected gear.
- Added `trade buff` and richer trade chat parsing, including stat specifiers for named trade requests.
- Added gear-management commands for `autoequip`, `autoequip debug`, `unequip everything`, and targeted `unequip <slot>`.
- Added proactive upgrade offers and future-frontier gear awareness so bots can suggest and offer upcoming upgrades, not just immediate swaps.
- Added separate trade windows per reservation group when bots hand over equips, making larger gear transfers easier to review.
- Added offline-owner support flows: bots can return to town safely, remain available through party/whisper control paths, and announce where they waited after the owner reconnects.
- Added a lightweight bot script/task framework and hooked in Kerning PQ automation, including scripted KPQ stage 1 behavior and stage 5 reward handling.
- Added new build coverage for Dark Knight, I/L Mage lines, and improved crossbow build support, plus a `change build` re-prompt flow.
- Added GM/admin tooling with `@expdebug` party EXP tracking and `!airshow <botname>` bot flight testing.

## 2. Changes / Improvements

- Reworked bot movement around an intent-driven architecture, giving movement, fidgeting, and physics a cleaner handoff and making future behavior easier to extend.
- Rebuilt swim-map handling with dedicated swim physics, stance dispatch, jump timing, and water-specific navigation behavior instead of forcing ground logic onto swim maps.
- Improved follow/combat coordination for cleric and support bots, including smarter heal movement and better follow-vs-combat decision rules.
- Improved ranged combat behavior with more stable repositioning, opportunity fire, cross-region coverage, anchored targeting, and less pointless oscillation.
- Improved grind distribution so groups spread across occupied regions more naturally and can recover back to a party anchor when displaced.
- Tightened bot command parsing and expanded command aliases across follow, grind, support, trading, supplies, build selection, and social actions.
- Routed bot replies through the triggering chat channel, so map chat, party chat, and whispers now receive replies in the same context instead of forcing everything through one path.
- Reworked auto-pot and resupply behavior: bots now prefer the cheapest valid recovery items, share recovery filters consistently, surface shop shortfalls, and recover more reliably from shop runs.
- Upgraded equip optimization and trade recommendation logic with weapon-slot lookahead, multi-slot lookahead, ring handling inside the optimizer, and a dynamic-programming recommendation path instead of older greedy selection.
- Improved gear reservation and offer logic so bots distinguish between current needs, future needs, own-class candidates, sibling-bot candidates, and disposable trade trash more accurately.
- Improved damage/range modeling for bots by sharing more of the real combat multipliers and using more realistic damage conditions during scoring and decision-making.
- Added better performance/diagnostic visibility through expanded slow logging, richer path logging, and more debug-oriented command output.

## 3. Bug Fixes

- Fixed a `lastlogin` null-guard issue.
- Fixed cleric bots casting Heal on non-undead mobs.
- Fixed cleric Heal formula, heal thresholding, heal range checks, and cast-animation stance handling.
- Fixed heal cast packets that could leave bots visually walking in place after healing.
- Fixed Arrow Bomb bot behavior.
- Fixed support buffs so bot skill buffs route through the correct special-move handling path.
- Fixed bot foothold id broadcasting so bot movement renders on the correct z-layer.
- Fixed invalid jump-attacks for bows, crossbows, wands, and staffs.
- Fixed invalid airborne ranged attacks.
- Fixed bot attack animation lock timing.
- Fixed bot ranged attack stalls.
- Fixed invisible ranged star-throw animations.
- Fixed bot retreat targeting and range reporting so hit-rate benchmarking uses the hardest relevant mob avoidability.
- Fixed follow-anchor navigation and ranged AoE anchor targeting.
- Fixed rope entry/exit handling using correct climbable-Y semantics.
- Fixed bots walking in place and fixed airborne-bounds handling on synthetic maps.
- Fixed stale drop-edge reuse, bottom-anchored wall collisions, underside/foothold collision behavior, and several down-jump edge cases.
- Fixed bot EXP-rate initialization on spawn so bot EXP handling matches human-player startup behavior.
- Fixed trade slot matching and suspended loot pickup / movement input while a trade is active.
- Fixed command and inventory replies that previously ignored channel context.
- Fixed owner-given gear tracking so temporary owner handoffs only affect reservation logic during the active trade session.
- Fixed fame-gated gear being treated as trash instead of preserved gear.
- Fixed GM fame handling so GMs respect normal daily/monthly fame cooldowns by default.
- Fixed equip optimizer issues around slot labels, ring self-equip paths, mastery accessory handling, non-weapon unlock probing, and overall vs top+pants combo scoring.
