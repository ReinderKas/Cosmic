# Bot Patch Notes v0.3.1

## 1. New Features

- Added `patrol`, `roam`, and `wander` bot commands for region-bounded grind behavior. Bots now patrol the navigation region under the owner when commanded instead of drifting freely.

## 2. Changes / Improvements

- Grind mode now allows short opportunistic loot detours when a drop is very close to the current combat path, reducing obvious ignored loot without turning grind into loot-chasing.

## 3. Bug Fixes

- Fixed Owner-offline town wait behavior.
- Fixed self-reserve logic for multi-skill weapon builds so bots can preserve both valid weapon families when they have both mastery lines.
- Fixed offer filtering so bots no longer offer incompatible weapon families to recipients such as Crossbowmen being offered bows.
- Fixed a self-reserve mismatch between equip logs and `trade equip` grouping: bag items are now classified against the full owned gear set, including currently equipped items.
- Fixed active patrol loot targeting when any inventory tab is full, preventing bots from pathing toward drops they cannot actually take.
