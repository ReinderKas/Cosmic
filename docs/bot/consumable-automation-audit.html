No-Donor And Backoff Behavior

Pot share requests

  • Shared owner-wide request cooldown: 30 seconds.
  • Failed HP request backoff: 10 minutes.
  • Failed MP request backoff: 10 minutes.
  • Request flag is still marked as sent when no donor is found or no donor qualifies.

Ammo share requests

  • Shared owner-wide request cooldown: 30 seconds.
  • Failed ammo request backoff: 10 minutes per owner and weapon type.
  • Request flag stays set until ammo recovers back to >= 500.

What happens if reserves keep dropping

  • Bot does not keep spamming requests while already in one low-supply episode.
  • If no donor exists, it can continue consuming reserves without another request until recovery or mode reset.
  • Eventually a hard stop may happen from HP pot exhaustion, ranged ammo exhaustion, or mage MP-pot exhaustion.

Donor qualification

  • Pot donor must have more than 300 matching pots to qualify.
  • Qualified pot donation amount is count / 3.
  • Ammo donor must have at least 500 matching ammo.
  • If donor also uses the same ammo type, donation is trimmed to preserve donor reserve.

Audit Notes

Observation Current Meaning
Low-pot request threshold and stop-farming threshold are different Correct. Pot requests start below 100, but HP-based grind stop waits until below 10 and current HP below 40%.
Low MP alone does not stop most jobs from farming Correct. Only mages have a hard follow-owner stop on total MP-recovery potion depletion.
No donor found while supplies continue decreasing The bot usually will not retry immediately. It enters a backoff window and the low-episode request flag remains set until reserves recover.
Shop automation is earlier than donor-share automation On a map that has a shop, resupply trigger is much higher: 400 pots and 4000 ammo.
